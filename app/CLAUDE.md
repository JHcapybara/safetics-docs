# TBM 앱 사용 설명서 — Claude 작업 지침서

이 폴더는 "TBM 모바일 앱(근로자용)" 사용 설명서의 원본과 도구 일체다.
누가 어떤 Claude 세션에서 열든, 이 지침을 **반드시** 따라 매뉴얼을 수정·빌드한다.
사용자는 대부분 비개발자(현장/운영)이므로 쉬운 말로 설명하고, 아래 절차는 네가 알아서 처리한다.

## 앱 ↔ 웹 역할 (혼동 주의)

이 매뉴얼은 **모바일 앱(참여·신고 전용)** 안내서다. TBM 발송·양식·근로자·민원 조치 등
관리는 **관리자 웹**(`../admin/` 매뉴얼)에서 한다. 문서 0장에 이 역할 구분표가 있으니 유지할 것.

## 파일 구조

| 경로 | 역할 |
|---|---|
| `index.html` | 매뉴얼 **원본. 모든 수정은 이 파일에만** 한다 |
| `TBM_앱_사용설명서_vX.XX.html` | 공유용 최종본(빌드 산출물). **직접 수정 금지**, 버전별로 쌓여 이력이 됨 |
| (저장소 루트 `app-latest.html`) | 빌드가 자동 생성하는 **고정 별칭** — 항상 최신 최종본으로 리다이렉트. 루트 index.html '최종본 보기'가 이걸 가리킴. 이 폴더가 아니라 **저장소 루트**에 생성됨. **직접 수정 금지** |
| `img/` | 스크린샷(01~18). `index.html`이 상대 경로로 참조 |
| `editor.html` | 비개발자용 GUI 편집기 (index.html을 열어 수정). admin과 동일본 |
| `tools/editor-find.js` | **두 편집기 공용** 찾기/하이라이트/치환 엔진. editor.html은 `<script src>`로 로드, 최종본은 build가 인라인 주입. admin과 동일본 — 수정 시 양쪽 동기화 |
| `tools/build.mjs` | 최종본 빌드 스크립트 (출력 파일명만 `TBM_앱_사용설명서_`로 지정, 나머지 로직은 admin과 동일) |
| `tools/import.mjs` | 수정된 최종본을 index.html로 역반영 (admin과 동일본, 무수정) |
| `readme.txt` | 사람용 안내서 |

## 절대 규칙

1. **수정은 `index.html`에만.** 최종본(`TBM_앱_사용설명서_*.html`)을 직접 고치지 말 것.
   예외: 사용자가 "수정된 최종본"을 들고 온 경우 → `node tools/import.mjs "<그 파일>"`로 index.html에 역반영부터.
2. **버전 규칙**: index.html 내용을 수정할 때마다 버전을 **+0.01** 올린다. 표기는 두 곳 — 반드시 동시 갱신:
   - `<!-- manual-version: X.XX -->` 주석
   - footer의 `<span id="docVersion">문서 버전 X.XX</span>`
3. **수정 후 항상 빌드**: 이 폴더에서 `node tools/build.mjs` → `TBM_앱_사용설명서_v{버전}.html` 생성. 구버전 파일은 지우지 않는다(이력).
4. **버전업 직후 사용자에게 이 메모를 대화로 보여줄 것** (index.html에는 넣지 않음):
   > 📌 app 문서 버전 X.XX로 업데이트·배포했습니다. PDF 페이지 전환 커스텀 설정할 것 있으면 요청해주세요 (형식: "2.2 앞에서").
5. **PDF 페이지 전환**: 사용자가 "2.2 앞에서"라고 하면 해당 항목의 `<article>`에 `pdf-break` 클래스를 추가한다
   (인쇄 CSS `article.pdf-break { break-before: page; }`가 이미 있음). **PDF 전용 — 웹 화면 표시가 바뀌면 안 됨.**
6. **용어/문구는 실제 앱 UI(i18n 한국어 리소스)와 100% 일치**시킨다. 혼용 용어는 첫 등장 시 병기
   (예: "건의사항(민원)", "위치 인증(Geo-Fence)", "외부 근로자(일용직)").
7. 문서 구조·디자인(사이드 목차, 검색, 스텝 번호, 주의/팁 박스, 이용자 유형 배지)은 기존 패턴을 유지한다.

## 검증 체크리스트 (빌드 후)

- 새 `TBM_앱_사용설명서_v{버전}.html`이 생성되었는지, 용량이 대략 3MB인지
- 빌드 로그에 "이미지 18개 내장" (누락 경고 없는지)
- 가능하면 브라우저(또는 Playwright)로 열어: 목차 이동·검색·이미지 표시 확인
- 섹션 번호 연속성, 깨진 앵커(#) 링크 없는지
- PDF 관련 수정이었다면: PDF로 저장해서 페이지 나눔 확인 (빈 페이지가 생기면 안 됨)

## 알아둘 것 (수정 시 함정 — tools/build.mjs에 이미 반영)

- **최종본에는 숨김 편집기가 내장**됨 (S+F+T 누른 채 "⚙ PDF 페이지 설정" 버튼 왼쪽 투명 버튼 3연클릭).
  빌드가 자동 주입하므로 별도 작업 불필요.
- `index.html`의 `<img>`에 빌드가 심는 `data-src` 속성은 역반영(import)에 필요 — 제거 금지.
- designMode 문서에서는 버튼/모달에 `contenteditable="false"`가 없으면 클릭이 안 먹고,
  파일 다운로드는 designMode를 끄고 **한 틱 뒤에** 실행해야 동작한다.
- 스크린샷 재촬영은 이 폴더만으로는 불가 — 앱 코드 레포(Expo 프로젝트)에서
  `npm run web` + `npm run screenshots:mock`(가짜 데모 데이터로 화면 렌더) 실행 후 img/ 복사, 다시 빌드.
- 인쇄 CSS 원칙: 장(chapter)은 항상 새 페이지, 항목(article)은 중간 분할 허용하되
  제목·표·이미지·박스는 잘리지 않게 보호. 항목 전체를 `break-inside: avoid`로 묶으면 빈 페이지가 생기니 금지.
- 모바일 스크린샷은 세로형이라 `figure.shot.mobile`(max-width 300px, 가운데 정렬), 2장 나란히는 `.duo`.
