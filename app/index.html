TBM 앱 사용 설명서

영업·마케팅·운영·총무·관리직 등 비개발 직군과 현장 근로자를 위한 TBM(Tool Box Meeting) 모바일 앱 사용 안내서입니다.

0. 핵심 용어 정리

꼭 알아야 할 용어/개념

관리자 웹 ↔ 모바일 앱, 역할이 다릅니다

구분관리자 웹 (PC)모바일 앱 (이 설명서)

파견다른 조직 현장의 TBM 대상으로 초대받는 것. 앱에서 수락하면 현장 선택 목록에 파견 현장이 추가됩니다(주황색 "파견" 배지).

조치 상태 3종건의·신고의 처리 단계: 미조치 → 조치중 → 조치 완료.

1. 시작하기

1.1 설치와 준비물 전체

앱을 설치하고 로그인에 필요한 것을 준비합니다.

이용자준비물
정규직회사 계정 이메일·비밀번호 (SafetyDesigner 계정과 동일). 계정이 없으면 사내 관리자(Admin)에게 등록을 요청하세요.
외부 근로자현장에 게시된 인증번호 + 본인 이름 + 휴대폰 번호. 인증번호는 관리자가 공장별로 발급해 현장에 게시합니다.

앱이 요청하는 권한

  • 위치 — 위치 인증(Geo-Fence) 서명과 현장 날씨 표시에 사용
  • 카메라·사진 — 위험원 보고, 작업 중지, 건의사항의 사진 첨부에 사용
  • 알림 — TBM 발송 Push 알림 수신에 사용
⚠️ 주의알림 권한을 거부하면 TBM이 발송돼도 Push 알림이 오지 않습니다. 앱은 정상 동작하지만 참여 시각을 놓치기 쉬우니 허용을 권장합니다.

1.2 로그인 — 정규직 (아이디 로그인) 정규직

이메일과 비밀번호로 로그인합니다.

  1. 앱을 열고 아이디 로그인 탭을 선택합니다.
  2. 아이디 칸에 이메일을 입력합니다. (입력 안내: "이메일을 입력하세요.")
    → 이메일/비밀번호는 SafetyDesigner 계정과 동일합니다.
  3. 비밀번호를 입력합니다.
  4. 다음부터 자동으로 로그인하려면 로그인 상태 유지를 체크합니다.
  5. 로그인을 누릅니다.
  6. 로그인 직후 생체인증 등록 안내가 뜰 수 있습니다 — 1.4 참고. (나중에로 건너뛰어도 됩니다)
💡 팁화면 상단의 KO/EN 버튼으로 로그인 화면 언어를 바꿀 수 있습니다. 로그인이 안 되면 4.4 로그인이 안 될 때를 보세요.

1.3 로그인 — 외부 근로자 (인증번호 로그인) 외부 근로자

현장에 게시된 인증번호와 이름·휴대폰 번호로 로그인합니다.

  1. 인증번호 로그인 탭을 선택합니다.
  2. 인증번호를 입력합니다. (현장 게시물 참고 — 자동으로 대문자 처리됩니다)
  3. 휴대폰 번호를 입력합니다. (안내: "01012345678 형식으로 입력해주세요.")
  4. 이름을 입력합니다.
  5. 로그인을 누릅니다.
  6. 본인 확인 단계가 이어집니다: 기기에 지문·얼굴 인식이 있으면 생체인증(최초 1회 등록 필수), 없으면 PIN 4자리를 설정·입력합니다.
⚠️ 주의외부 근로자는 보안을 위해 로그인할 때마다 본인 확인(생체인증 또는 PIN)을 거치며, 자동 로그인이 되지 않습니다. PIN을 잊었다면 "PIN을 잊으셨나요?" → 안내에 따라 관리자에게 인증 정보 초기화를 요청하세요. 초기화 후 다시 로그인하면 새 PIN을 설정할 수 있습니다.
💡 팁인증번호가 맞지 않으면 관리자가 번호를 갱신했을 수 있습니다. 현장에 게시된 최신 번호를 확인하세요.

1.4 생체인증·자동 로그인 전체

다음 로그인부터 지문·얼굴 인식으로 빠르게 들어갑니다.

정규직

  1. 이메일 로그인 직후 뜨는 안내에서 등록을 누릅니다. (안내 문구: "다음부터 생체인증(지문·얼굴)으로 빠르게 로그인하시겠습니까?")
  2. 기기의 지문·얼굴 인식을 완료하면 "생체인증 로그인이 등록되었습니다." 메시지가 표시됩니다.
  3. 다음부터 로그인 화면에 생체인증 로그인 버튼이 나타납니다 — 누르면 비밀번호 없이 로그인됩니다.
  4. 로그인할 때 로그인 상태 유지를 체크해 두면 앱을 다시 열 때 자동으로 로그인됩니다.

외부 근로자

  • 생체인증(또는 PIN)이 필수입니다. 매 로그인 시 본인 확인을 거치며, 자동 로그인은 지원하지 않습니다.
💡 팁휴대폰을 바꿨거나 생체인증이 계속 실패하면 관리자에게 인증 정보 초기화를 요청한 뒤 다시 등록하세요.

1.5 화면 구성 익히기 전체

홈 화면과 오른쪽 메뉴만 알면 모든 기능에 닿을 수 있습니다.

홈 화면 (위에서 아래로)

  • 상단 헤더: 현재 현장 이름 (여러 현장 소속이면 ▼ 표시 — 탭하여 현장 선택) + ☰ 메뉴 버튼
  • 현장 날씨 카드: 현재 기온·풍속·습도, 최저/최고/강수확률. 새로고침 버튼 제공
  • 오늘의 TBM 카드: 오늘 참여할 TBM — 탭하면 바로 시작 (2.1)
  • 긴급 카드 3종 (좌우로 넘김): 위험원 보고 / 긴급 SOS 전송 / 작업 중지 요청 (4장)
  • 건의사항 요약 카드: 조치 완료 / 조치중 / 미조치 건수 + 최근 건의. 전체 건의사항 보기로 이동

오른쪽 메뉴 (☰ 버튼)

  • 홈(대시보드) / Tool Box Meeting(작업전 안전회의) / 건의사항(의견·요청) / 설정(계정·알림)
  • 언어 — 한국어(기본) / English (3.5)
  • 로그아웃 (3.6)

1.6 현장 선택 (여러 현장 소속일 때) 정규직

여러 현장(사업장)에 소속됐거나 파견 중이라면, 보고 있는 현장을 전환합니다.

  1. 홈 상단의 현장 이름 ▼을 탭합니다. (현장이 하나뿐이면 ▼가 없고 전환할 필요도 없습니다)
  2. 현장 선택 목록에서 원하는 현장을 탭합니다. 현재 현장에는 ✓ 표시가 있습니다.
  3. 파견으로 소속된 현장에는 주황색 파견 배지가 붙습니다.
💡 팁현장을 바꾸면 오늘의 TBM · TBM 기록 · 건의사항이 모두 선택한 현장 기준으로 바뀝니다. "오늘 TBM이 안 보여요"의 흔한 원인이 다른 현장을 보고 있는 경우입니다. 설정 화면에서도 현장을 바꿀 수 있습니다. 외부 근로자는 인증번호의 현장 한 곳으로 고정됩니다.

2. 주요 기능 | 매일 하는 일 — TBM 참여

2.1 오늘의 TBM 시작하기 ★ 홈 또는 Tool Box Meeting 탭 전체

오늘 발송된 TBM을 열어 참여를 시작합니다. 매일 가장 자주 쓰는 기능입니다.

  1. 홈의 오늘의 TBM 카드를 탭하거나, 메뉴에서 Tool Box Meeting으로 이동해 TODAY 카드의 TBM 시작하기를 누릅니다.
  2. 요약 화면에서 오늘 내용을 확인합니다: 작업 책임자 / 현장 / 참석 인원, 그리고 체크리스트 구성 — 주의사항·개인안전보호구(PPE)·오늘의 특별 공지별 건수.
  3. 하단 시작하기를 누르면 체크리스트가 시작됩니다 (2.2).
⚠️ 주의시작 시각 전에는 버튼 대신 "{시각}부터 참여할 수 있습니다"가 표시되고 눌리지 않습니다. 오늘 TBM이 없으면 "오늘 등록된 TBM 일정이 없습니다."가 표시됩니다 — 발송 여부는 관리자에게 확인하세요. 여러 현장 소속이라면 현장 선택이 맞는지 먼저 확인하세요.

2.2 체크리스트 응답하기 ★ 전체

체크리스트 항목을 한 장씩 읽고 "확인하였음" 또는 "이슈 있음"으로 응답합니다.

  1. 카드에 표시된 항목을 읽습니다. 상단에 주의사항 1 / 3처럼 분류와 순번이, 항목에 따라 위험인자 / 유해요인 태그가 표시됩니다.
  2. 문제 없으면 확인하였음(파란색)을 누릅니다. "확인 완료" 표시가 잠깐 나타나고 다음 항목으로 넘어갑니다.
  3. 현장 상태가 항목과 다르거나 문제가 있으면 이슈 있음(회색)을 누릅니다. "이슈 보고됨"이 표시되고, 해당 항목이 관리자에게 TBM 이슈로 자동 접수됩니다.
  4. 실수로 잘못 눌렀다면 이전으로 돌아가 다시 응답할 수 있습니다. 응답해야 다음이 활성화됩니다.
  5. 모든 항목에 응답하면 참여 완료 단계로 넘어갑니다 — 서명 방식에 따라 2.3 자필 서명 / 2.4 위치 인증 / 확인만(버튼 한 번).
💡 팁"이슈 있음"으로 보고한 내용은 건의사항에서 "[TBM 이슈]"로 확인하고 조치 이력을 추적할 수 있습니다 (3.2).

2.3 참여 완료 — 자필 서명 전체

서명 방식이 "자필 서명"인 TBM은 화면에 손가락으로 서명해 참여를 마칩니다.

  1. 응답 요약(전체 / 확인 / 이슈 건수)을 확인합니다. 수정하려면 목록으로 돌아가기로 되돌아갑니다.
  2. 서명 영역("서명하기")에 손가락으로 서명합니다. 잘못 그렸으면 다시 서명으로 지우고 다시 합니다.
  3. TBM 완료하기를 누릅니다. 이슈를 보고한 항목이 있으면 버튼이 이슈 보고 및 서명 제출로 표시됩니다.
  4. 전송 중에는 "전송 중…"이 표시됩니다. 완료되면 TBM 목록으로 돌아가고 오늘 TBM이 완료 상태가 됩니다.
⚠️ 주의서명을 하지 않으면 완료 버튼 자리에 "서명을 해주세요."가 표시됩니다. "서명 전송 실패" 안내가 뜨면 네트워크 상태를 확인한 뒤 다시 시도하세요 — 제출 전이므로 참여 기록은 아직 남지 않은 상태입니다.

2.4 참여 완료 — 위치 인증(Geo-Fence) 전체

서명 방식이 "Geo-Fence"인 TBM은 현장 GPS 반경 안에 있는지 확인해 참여를 마칩니다.

  1. 체크리스트 응답이 끝나면 자동으로 "위치 확인 중…"이 진행됩니다. (위치 권한 허용 필요)
  2. "위치 인증 완료 — 현장 반경 내 위치가 확인되었습니다."가 표시되면 TBM 완료하기를 누릅니다.
  3. 실패 시 "현장 반경 밖에 있습니다. (거리 m / 허용 m)"가 표시됩니다 — 현장 안으로 이동한 뒤 다시 시도를 누르세요.
💡 팁위치 권한·GPS 오류가 계속되면 4.5 위치 인증이 안 될 때를 보세요.

2.5 지난 TBM 기록 확인 Tool Box Meeting > 이전 기록 전체

과거에 참여한(또는 놓친) TBM 내용을 다시 봅니다.

  1. 메뉴에서 Tool Box Meeting으로 이동해 이전 기록 목록을 봅니다.
  2. 상단 필터 탭으로 거릅니다: 전체 / 완료 / 미완료 / 예정 (건수 표시).
  3. 기록을 탭해 상세를 엽니다. 항목별로 확인 완료 / 이슈 있음 응답, 특별 공지, 그리고 참석 확인 서명(자필 서명 이미지, 또는 "위치 인증으로 참여 완료")을 볼 수 있습니다.
💡 팁서명 칸에 "서명 기록이 없습니다. 미참여 또는 미완료일 수 있습니다."가 보이면 그날 TBM에 참여하지 않은 것입니다. 예정 상태의 기록은 아직 열 수 없습니다.

3. 부가 기능 | 가끔 하는 일

3.1 건의사항 작성하기 건의사항 > ✏ 전체

현장 개선 요청이나 의견을 사진과 함께 등록합니다.

  1. 메뉴에서 건의사항으로 이동해 오른쪽 아래 ✏ 버튼을 누릅니다.
  2. 제목을 입력합니다. (필수, "건의 제목을 입력해 주세요")
  3. 내용을 입력합니다. ("개선이 필요한 점과 현장 상황을 자세히 작성해 주세요")
  4. 필요하면 사진 첨부 — 카메라로 촬영 또는 앨범에서 선택, 최대 3장.
  5. 등록을 누릅니다. 등록된 건의는 목록에서 미조치 상태로 시작합니다.
💡 팁위험 요소를 발견했다면 일반 건의 대신 홈의 위험원 보고 카드를 쓰세요 (4.1) — 관리자에게 더 눈에 띄게 전달됩니다.

3.2 건의사항 조회·답글 달기 건의사항 전체

내 건의·신고의 조치 진행 상황을 확인하고 관리자와 답글로 소통합니다.

  1. 건의사항 목록에서 검색·필터 칩으로 원하는 건을 찾습니다. 상태는 미조치 / 조치중 / 조치 완료로 표시됩니다.
  2. 건을 탭해 상세를 엽니다. 위험원 보고는 "[위험원]", SOS는 "[긴급] SOS 요청", TBM 이슈는 "TBM 이슈 발생"으로 제목이 붙습니다.
  3. 조치 이력에서 관리자의 답글·상태 변경 기록을 확인합니다. (아직 없으면 "등록된 조치 이력이 없습니다.")
  4. 하단 입력창("답글을 입력하세요")에 내용을 쓰고 전송합니다. 내가 쓴 답글은 수정·삭제할 수 있습니다.
⚠️ 주의조치 완료된 건은 답글 입력창이 닫힙니다. 추가로 할 말이 있으면 새 건의를 등록하세요.

3.3 파견 초대 수락/거절 정규직

다른 조직 현장에서 온 파견 초대를 수락하거나 거절합니다.

  1. 파견 초대가 오면 로그인 직후 홈에 "파견 초대" 창이 자동으로 뜹니다. 나중에 처리하려면 설정 화면의 파견 초대 (N) 섹션에서도 볼 수 있습니다.
  2. 초대 카드에서 초대한 현장·조직, 기간(시작일 없으면 "즉시", 종료일 없으면 "무기한"), 메모를 확인합니다.
  3. 수락 또는 거절을 누릅니다.
  4. 수락하면 현장 선택 목록에 파견 현장이 파견 배지와 함께 추가되고, 그 현장의 TBM에 참여할 수 있습니다.
💡 개념파견 초대는 관리자가 웹에서 보냅니다. 파견이 종료되면(관리자 종료 또는 기간 만료) 현장 목록에서 자동으로 사라집니다. 외부 근로자에게는 파견 기능이 없습니다.

3.4 알림 설정 설정 전체

Push 알림 수신 여부를 조절합니다.

  1. 메뉴에서 설정으로 이동합니다. 내 이름·소속 현장·권한을 확인할 수 있습니다.
  2. App Push 알림 받기 — 앱 전체 Push 알림을 켜고 끕니다.
  3. TBM 등록 알림 받기 — TBM 발송 알림만 따로 켜고 끕니다.
⚠️ 주의앱 안의 토글을 켜도 휴대폰 설정에서 앱 알림 자체를 꺼두면 알림이 오지 않습니다. 알림이 안 오면 휴대폰 설정 > 알림에서 이 앱의 알림 허용 여부를 먼저 확인하세요.

3.5 언어 변경 (한국어/English) 메뉴 > 언어 전체

앱 표시 언어를 바꿉니다.

  1. ☰ 메뉴를 열고 언어를 탭합니다.
  2. 한국어(기본) 또는 English를 선택합니다 — 즉시 적용됩니다.
💡 팁로그인 화면에서는 상단 KO/EN 버튼으로 바꿉니다. 처음 설치하면 휴대폰 언어를 따라갑니다.

3.6 로그아웃·계정 삭제 전체

계정에서 나가거나, 계정을 완전히 삭제합니다.

로그아웃

  1. ☰ 메뉴 하단의 로그아웃을 누릅니다.
  2. 확인 창("로그아웃 하시겠습니까?")에서 로그아웃을 누릅니다. 자동 로그인도 함께 해제됩니다.

계정 삭제

  1. 설정 화면 맨 아래 계정 삭제 섹션에서 계정 삭제를 누릅니다.
  2. 확인 창("정말 계정을 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다.")에서 계정 삭제를 누릅니다.
  3. "계정이 삭제되었습니다." 확인 후 로그인 화면으로 돌아갑니다.
🚫 위험계정을 삭제하면 모든 데이터가 제거되며 복구할 수 없습니다. 회사를 옮기거나 서비스를 완전히 그만둘 때만 사용하세요. 단순히 기기를 바꾸는 경우라면 로그아웃만 하면 됩니다.

4. 문제 생겼을 때 | 긴급 신고와 문제 해결

4.1 위험원 보고 ★ 홈 > 위험원 보고 카드 전체

현장에서 발견한 위험 요소를 관리자에게 즉시 알립니다.

  1. 홈의 긴급 카드에서 위험원 보고(RISK, "탭하여 보고")를 탭합니다.
  2. 제목과 내용에 위험 위치·상황을 구체적으로 적습니다. ("미조치 위험원이 있나요? 관리자에게 즉시 알립니다.")
  3. 현장 사진을 첨부합니다 (최대 3장 — 사진이 있으면 조치가 훨씬 빨라집니다).
  4. 등록을 누릅니다. 건의사항 목록에 [위험원] 표시로 등록되고, 조치 이력을 추적할 수 있습니다 (3.2).

4.2 긴급 SOS ★ 홈 > SOS 카드 전체

부상·조난 등 긴급 상황에서 관리자에게 구조 신호를 보냅니다.

  1. 홈의 긴급 카드에서 SOS를 탭합니다.
  2. 확인 창 "긴급 SOS를 관리자에게 전송하시겠습니까?"에서 예, 전송합니다를 누릅니다. (실수로 눌렀다면 아니오)
  3. SOS 화면이 열리며 사이렌 소리와 진동이 울려 주변에 상황을 알립니다. "SOS 전송 중..." 상태가 표시됩니다.
  4. 상황이 정리되면 SOS 종료를 누릅니다 — 이때 관리자에게 신호가 전송되고 "SOS 전송 완료 — 관리자에게 긴급 신호가 전달되었습니다."와 전송 시각이 표시됩니다.
  5. 홈으로 돌아가기로 복귀합니다.
🚨 중요관리자에게는 "[긴급 SOS] 현장에서 긴급 구조 요청이 발생했습니다. 즉시 확인 바랍니다." 메시지가 전달되며, 웹 관리자 화면에서 최우선 알림으로 표시됩니다. 훈련·테스트 목적이라면 반드시 관리자와 사전에 협의하세요.

4.3 작업 중지 요청 홈 > 작업 중지 요청 카드 전체

위험해서 작업을 계속할 수 없을 때, 관리자에게 즉시 작업 중지를 요청합니다.

  1. 홈의 긴급 카드에서 작업 중지 요청(STOP, "탭하여 요청")을 탭합니다.
  2. 사유를 선택합니다 — "사유를 선택해 주세요"를 탭하면 목록이 열립니다 (예: "추락 위험: 안전벨트 미착용, 안전난간 부실" 등 24종). 맞는 항목이 없으면 직접 입력을 골라 사유를 적습니다.
  3. 상세 내용 (선택)에 "관리자가 조치를 취할 수 있는 자세한 위치 및 내용"을 적습니다.
  4. 필요하면 사진 첨부 (선택) — 최대 3장.
  5. 작업 중지 요청 전송을 누릅니다. "관리자에게 작업 중지 요청이 전달되었습니다." 확인 창이 뜹니다.
💡 팁전송된 요청은 건의사항 목록에서 조치 상태(미조치→조치중→조치 완료)를 추적할 수 있습니다. 관리자 웹에서는 ⚠️ 작업중지 최우선 알림으로 표시됩니다.

4.4 로그인이 안 될 때 전체

로그인 화면 오류 메시지별 원인과 해결 방법입니다.

화면 메시지원인과 해결
"아이디 또는 비밀번호가 올바르지 않습니다."입력 오류. 대소문자·공백 확인 후 재시도. 이메일/비밀번호는 SafetyDesigner 계정과 동일 — 비밀번호를 잊었다면 Safetics 홈페이지에서 재설정.
"소속된 조직이 없습니다"계정이 아직 조직에 등록되지 않음. 사내 관리자(Admin)에게 등록 요청.
"TBM 서비스 계약이 만료되었습니다"회사의 TBM 계약 문제. 사내 담당자에게 문의.
"TBM 권한이 없습니다"계정에 TBM 이용 권한이 없음. 관리자에게 권한 확인 요청.
"인증번호가 올바르지 않습니다"인증번호 오타 또는 관리자가 번호를 갱신함. 현장 게시 최신 번호 확인.
"등록된 근로자를 찾을 수 없습니다"이름·휴대폰 번호가 등록 정보와 다름. 관리자에게 등록 정보 확인 요청.
"본인 확인에 실패했습니다" / "생체인증 로그인에 실패했습니다."지문·얼굴 인식 실패. 다시 시도하거나, 기기 변경 시 관리자에게 인증 정보 초기화 요청.
"인증이 만료되었습니다. 다시 로그인해주세요"오랜만에 접속해 세션이 만료됨. 다시 로그인하면 됩니다.
"인터넷 연결을 확인해주세요"네트워크 문제. Wi-Fi/데이터 연결 확인 후 재시도.

4.5 위치 인증(Geo-Fence)이 안 될 때 전체

위치 인증 실패 메시지별 대처 방법입니다.

화면 메시지원인과 해결
"위치 권한이 필요합니다. 설정에서 위치 권한을 허용해 주세요."앱 위치 권한 거부 상태. 휴대폰 설정 > 앱 > 권한에서 위치를 허용한 뒤 다시 시도.
"GPS 신호를 받을 수 없습니다. 실외에서 다시 시도해 주세요."실내·지하 등 GPS 음영 지역. 하늘이 보이는 곳으로 이동 후 재시도.
"현장 반경 밖에 있습니다. (거리 m / 허용 m)"현재 위치가 현장 GPS 반경 밖. 현장 안으로 이동 후 다시 시도. 현장 안인데도 반복되면 관리자에게 GPS 핀·반경 설정 확인을 요청하세요.
"현장 범위를 벗어났습니다"제출 시점 위치 검증 실패. 위와 동일하게 현장 내에서 재시도.
⚠️ 주의휴대폰에 위치 변조(가짜 GPS) 앱이 설치돼 있으면 인증이 거부될 수 있습니다.

부록

A. 자주 묻는 질문 (FAQ)

Q. 오늘 TBM이 안 보여요.
A. ① 여러 현장 소속이라면 홈 상단 현장 선택이 맞는지 확인하세요. ② 관리자가 아직 발송하지 않았을 수 있습니다 — 발송 여부를 관리자에게 확인하세요. ③ "{시각}부터 참여할 수 있습니다"가 보이면 시작 시각 전입니다.
Q. "TBM 시작하기" 버튼이 안 눌러져요.
A. 시작 시각 전이면 버튼이 비활성화됩니다. 표시된 시각 이후에 다시 시도하세요.
Q. 체크리스트에서 실수로 "이슈 있음"을 눌렀어요.
A. 제출 전이라면 이전으로 돌아가 확인하였음으로 다시 응답하면 됩니다. 이미 제출했다면 관리자에게 알려주세요.
Q. 서명을 잘못 그렸어요.
A. 제출 전이라면 다시 서명으로 지우고 다시 서명하세요. TBM 완료하기를 누른 뒤에는 수정할 수 없습니다.
Q. 이미 참여했는데 또 하라고 나와요 / "이미 참여 완료된 TBM입니다"가 떠요.
A. 이미 완료된 TBM은 다시 참여할 수 없습니다. 목록에서 상태가 완료인지 확인하세요. 화면이 이전 상태로 보이면 목록을 아래로 당겨 새로고침하세요.
Q. 인증번호가 안 돼요. (외부 근로자)
A. 관리자가 인증번호를 갱신하면 이전 번호는 무효가 됩니다. 현장에 게시된 최신 번호를 확인하고, 이름·휴대폰 번호가 등록 정보와 같은지 확인하세요.
Q. 파견 초대가 안 보여요.
A. 초대는 로그인 직후 홈 모달과 설정 > 파견 초대 섹션에 표시됩니다. 안 보이면 초대가 취소·만료됐을 수 있으니 초대한 쪽 관리자에게 상태를 확인해 달라고 하세요.
Q. Push 알림이 안 와요.
A. ① 설정 화면의 App Push 알림 받기·TBM 등록 알림 받기 토글 확인. ② 휴대폰 설정 > 알림에서 이 앱의 알림 허용 확인. ③ 그래도 안 오면 로그아웃 후 다시 로그인하세요.
Q. 휴대폰을 바꿨어요.
A. 새 기기에서 다시 로그인하면 됩니다. 생체인증은 새 기기에서 다시 등록하세요. 외부 근로자가 본인 확인(생체/PIN)에 계속 실패하면 관리자에게 인증 정보 초기화를 요청하세요.

C. 용어집

TBM (Tool Box Meeting)
작업 전 실시하는 단시간 안전 미팅. 관리자 웹에서 발송하고, 이 앱에서 체크리스트·서명으로 참여합니다.
체크리스트
TBM 1건의 확인 항목 묶음. 주의사항 / 개인안전보호구(PPE) / 오늘의 특별 공지로 구성.
주의사항
오늘 작업에서 지켜야 할 안전 확인 항목. 위험인자·유해요인 태그가 붙을 수 있습니다.
개인안전보호구 (PPE)
안전모·안전화·보안경 등 개인 보호 장비 착용 확인 항목.
오늘의 특별 공지
관리자가 그날 특별히 전달하는 공지 (예: 우천 시 주의사항).
확인하였음 / 이슈 있음
체크리스트 항목별 응답. "이슈 있음"은 관리자에게 TBM 이슈로 자동 접수됩니다.
위치 인증 (Geo-Fence)
현장 GPS 핀 + 반경으로 만든 가상 울타리 안에 있는지 확인하는 서명 방식.
자필 서명
화면에 손가락으로 직접 서명하는 참여 확인 방식.
인증번호
외부 근로자가 로그인할 때 입력하는 공장(현장)별 코드. 관리자가 발급·갱신해 현장에 게시합니다.
외부 근로자 (일용직)
정식 계정 없이 인증번호 + 이름 + 휴대폰 번호로 참여하는 근로자.
파견
다른 조직 현장의 TBM 대상으로 초대받는 것. 수락하면 현장 선택 목록에 "파견" 배지와 함께 추가됩니다.
건의사항
위험원 보고·작업 중지·SOS·건의·TBM 이슈가 모이는 소통 창구. 조치 이력(답글)로 진행 상황을 추적합니다.
위험원
현장에서 발견한 위험 요소. "위험원 보고"로 관리자에게 신고합니다.
조치 이력
건의·신고에 대한 관리자·작성자의 답글과 상태 변경 기록.
현장 / 사업장 / 공장
조직 단위. 화면에 따라 "현장", "사업장", "공장"으로 표시되며 대체로 같은 단위를 가리킵니다.

D. 오류 메시지 사전

메시지원인해결
"아이디 또는 비밀번호가 올바르지 않습니다."로그인 정보 오류재입력. 반복되면 SafetyDesigner 비밀번호 재설정.
"소속된 조직이 없습니다"계정이 조직 미등록사내 Admin에게 등록 요청.
"TBM 서비스 계약이 만료되었습니다"회사 계약 문제사내 담당자 문의.
"TBM 권한이 없습니다"계정에 TBM 권한 없음관리자에게 권한 확인 요청.
"인증번호가 올바르지 않습니다"번호 오타 또는 갱신됨현장 게시 최신 번호 확인.
"등록된 근로자를 찾을 수 없습니다"이름·휴대폰 번호 불일치관리자에게 등록 정보 확인.
"인증이 만료되었습니다. 다시 로그인해주세요"세션 만료다시 로그인.
"본인 확인에 실패했습니다" / "PIN 확인에 실패했습니다"생체인증/PIN 실패재시도. 반복 시 관리자에게 인증 정보 초기화 요청.
"TBM 세션을 찾을 수 없습니다"삭제됐거나 잘못된 접근목록 새로고침 후 다시 진입.
"이미 참여 완료된 TBM입니다"중복 참여 시도참여 완료 상태 — 추가 조치 불필요.
"현장 범위를 벗어났습니다" / "현장 반경 밖에 있습니다. (거리/허용)"위치 인증 실패현장 안으로 이동 후 다시 시도 (4.5).
"위치 권한이 필요합니다. 설정에서 위치 권한을 허용해 주세요."위치 권한 거부휴대폰 설정에서 위치 권한 허용.
"GPS 신호를 받을 수 없습니다. 실외에서 다시 시도해 주세요."GPS 음영 지역실외 이동 후 재시도.
"서명 전송 실패 — 서명 이미지를 업로드하지 못했습니다..."네트워크 불안정연결 확인 후 다시 TBM 완료하기.
"이미 보고된 이슈입니다"같은 항목 이슈 중복 보고이미 접수됨 — 추가 조치 불필요.
"민원 등록에 실패했습니다"일시적 오류/네트워크연결 확인 후 재시도.
"인터넷 연결을 확인해주세요"오프라인Wi-Fi/데이터 연결 후 재시도.
"날씨 정보를 가져오지 못했습니다."날씨 조회 실패날씨 카드의 새로고침 버튼. 기능 이용에는 지장 없음.
"처리에 실패했습니다"일시적 오류재시도. 반복되면 문의 채널로 화면 캡처와 함께 신고.
