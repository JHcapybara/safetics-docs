====================================================
 TBM 관리자 사용 설명서 — 폴더 안내 (docs/TBM-Admin-manual/)
====================================================

■ 가장 쉬운 사용법 — Claude에게 맡기기 (권장)
----------------------------------------------------
  이 폴더에는 Claude(AI) 작업 지침서(CLAUDE.md)가 들어 있어서,
  Claude Code가 설치된 PC라면 누구나 아래처럼 쓰면 됩니다.
  1. 이 폴더에서 Claude Code를 연다
  2. 자유롭게 요청한다. 예:
       "2.4 참여 현황 설명에 ~내용 추가해줘"
       /manual-update 3.2 제목을 '양식 편집·삭제'로 바꿔줘
  3. Claude가 알아서: 수정 → 버전 +0.01 → 최종본 재생성 → 결과 보고
     (빌드 도구 Node.js가 없으면 설치 안내/설치까지 Claude가 처리)

■ 파일/폴더 역할
----------------------------------------------------
CLAUDE.md
  - Claude(AI)용 작업 지침서. 버전 규칙·빌드 절차·함정 노트가 담겨 있어
    어떤 Claude 세션이든 동일한 방식으로 작업하게 만든다. 삭제 금지.

.claude/commands/manual-update.md
  - Claude Code에서 쓰는 /manual-update 명령 정의.

tools/
  - build.mjs: 공유용 최종본 생성 (node tools/build.mjs)
  - import.mjs: 수정된 최종본을 index.html로 역반영 (node tools/import.mjs "<파일>")

index.html
  - 매뉴얼 "원본". 모든 수정은 이 파일에만 한다.
  - 이미지를 img/ 폴더에서 상대 경로로 불러오므로 이 파일만 단독으로 공유하면 스크린샷이 안 보인다. (공유용 아님)
  - 파일 안의 <!-- manual-version: X.XX --> 주석과 하단 footer의 "문서 버전 X.XX"가 현재 버전 표기.

editor.html
  - 비개발자용 매뉴얼 편집기. 코딩 없이 화면 보면서 글 수정.
  - 반드시 index.html을 열어서 수정할 것.
    (TBM_사용설명서_vX.XX.html을 열어 수정하면 다음 빌드 때 사라짐)

TBM_사용설명서_vX.XX.html
  - "공유용 최종본". 빌드 명령으로 자동 생성되는 결과물.
  - 이미지가 파일 안에 내장되어 있어 이 파일 하나만 메일/메신저로 보내면 어디서든 열림 (~1.5MB).
  - 직접 수정 금지. 버전별로 파일이 쌓여서 이력 아카이브 역할.

img/
  - 매뉴얼 스크린샷 원본 (01-login.png ~ 12-settings.png).
  - index.html이 참조. 지우면 원본 매뉴얼에서 이미지가 깨짐.

TOC.md
  - 최초 기획 시 작성한 목차 초안 (참고용, 없어도 동작에 지장 없음).

readme.txt
  - 이 파일.


■ 사용 방법
----------------------------------------------------
[매뉴얼 보기]
  index.html 또는 TBM_사용설명서_vX.XX.html을 더블클릭 (브라우저로 열림)
  - 왼쪽 목차 클릭으로 이동, 상단 검색창으로 본문 검색 가능

[PDF로 저장]
  매뉴얼 우측 상단 "PDF로 저장 / 인쇄" 버튼 → 대상: PDF로 저장
  - 페이지 나눔은 PDF 전용으로 설정되어 있음 (웹 화면과 무관)

[내용 수정하기 — 비개발자도 가능]
  1. editor.html 더블클릭 (Chrome 또는 Edge 권장)
  2. "매뉴얼 열기" → 같은 폴더의 index.html 선택
  3. 고칠 문장 클릭 후 그냥 타이핑 (실수하면 Ctrl+Z)
     - 글자 선택 시 상단에 서식 메뉴 표시: 문단 종류/크기/굵게/색/지우기
     - 같은 단어 전체 교체: "용어 일괄 치환" 버튼
  4. 저장 (Ctrl+S) → index.html에 바로 반영
     - 첫 저장 때 원본 백업본이 자동 다운로드됨

[수정 후 공유용 최종본 만들기]
  가장 쉬움: Claude에게 맡기기 (맨 위 "가장 쉬운 사용법" 참고)
  직접 할 경우:
  1. index.html의 버전 두 곳을 +0.01 올린다
     (<!-- manual-version: X.XX --> 주석, footer의 "문서 버전 X.XX")
  2. 이 폴더에서 터미널 실행:  node tools/build.mjs
     (저장소에서는 npm run manual:build 도 동일)
  3. TBM_사용설명서_v{새버전}.html 생성됨 → 이 파일을 공유

[스크린샷 다시 찍기 — 앱 UI가 바뀌었을 때, 개발 환경 필요]
  1. 터미널 1:  $env:VITE_DEV_SKIP_AUTH='false'; npm run dev
  2. 터미널 2:  npm run screenshots:mock
     - 백엔드 서버 불필요. 가짜 데모 데이터로 실제 화면을 렌더해서 img/ 폴더에 11장 자동 저장 (실데이터 노출 없음)
  3. 이후 "공유용 최종본 만들기" 다시 실행


■ 숨김 편집기 (관리자용 — 일반 배포 시 이 항목은 안내하지 않음)
----------------------------------------------------
  최종본(TBM_사용설명서_vX.XX.html) 자체에 편집 기능이 숨겨져 있다.
  - 진입: 키보드 S+F+T를 누른 채, 우측 상단 "PDF로 저장/인쇄" 버튼
    바로 왼쪽의 빈 공간(투명 버튼)을 1.5초 안에 3번 연속 클릭
  - 기능: editor.html과 동일 (문장 클릭 수정, 서식, 용어 일괄 치환, Ctrl+Z)
  - 저장(Ctrl+S): 버전이 자동으로 +0.01 되고, 새 버전 파일명으로 저장 제안
    (Chrome/Edge는 첫 저장 때만 위치 지정, 이후 조용히 저장. 그 외 브라우저는 다운로드)
  - 편집 UI는 저장본에 포함되지 않으며, 저장본도 다시 숨김 편집 가능
  - 수정된 최종본을 원본(index.html)에 반영하려면:
      npm run manual:import -- "<수정된 최종본.html>"
    (기존 index.html은 index.html.bak으로 자동 백업, 이후 manual:build로 재생성)

■ 규칙 요약
----------------------------------------------------
  - 수정은 index.html에만 / 공유는 TBM_사용설명서_vX.XX.html만
  - 수정할 때마다 버전 +0.01
  - PDF 페이지 나눔 설정: 매뉴얼 우측 상단 "⚙ PDF 페이지 설정" 버튼 → 섹션별 체크 토글
    (체크한 항목부터 PDF 새 페이지 시작, 웹 화면에는 영향 없음)
    · 매뉴얼 화면에서 토글 → 그 자리에서 인쇄하면 즉시 적용 (일회성)
    · 파일에 영구 저장하려면: editor.html의 "📄 PDF 전환" 버튼으로 설정 후 저장,
      또는 최종본의 숨김 편집 모드에서 ⚙ 설정 후 저장
