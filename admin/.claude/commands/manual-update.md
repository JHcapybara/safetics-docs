TBM 관리자 사용 설명서 수정 요청이다. 이 폴더(또는 docs/TBM-Admin-manual)의 CLAUDE.md 지침을 먼저 읽고, 아래 요청을 표준 작업 절차대로 처리하라.

수정 요청: $ARGUMENTS

처리 순서 (누락 금지):
1. 요청이 모호하면 작업 전에 짧게 되물어 확정한다. (예: 어느 섹션인지, 문구 원문)
2. index.html에서 수정한다. 최종본 파일은 절대 직접 수정하지 않는다.
3. 버전을 +0.01 올린다 — `<!-- manual-version -->` 주석과 footer `#docVersion` 두 곳 모두.
4. Node.js 확인 후 `node tools/build.mjs` 실행 (없으면 CLAUDE.md의 Node 설치 절차 수행).
5. 검증 체크리스트 수행 (새 버전 파일 생성, 이미지 11개 내장, 링크·번호 이상 없음).
6. 결과 보고: 무엇이 바뀌었는지 + 새 버전 파일명. 그리고 다음 메모를 반드시 출력:
   "📌 문서 버전 X.XX로 업데이트했습니다. PDF 페이지 전환 커스텀 설정할 것 있으면 추가로 요청해주세요 (형식: '2.2 앞에서')."

특수 케이스:
- 요청이 "N.N 앞에서" 형태면: 해당 섹션 article에 pdf-break 클래스만 추가하는 PDF 페이지 전환 작업이다 (화면 표시 불변).
- 사용자가 수정된 최종본 파일을 언급하면: `node tools/import.mjs "<파일>"`로 index.html에 역반영한 뒤 위 절차를 계속한다.
- 스크린샷 교체 요청이면: 이 폴더만으로는 불가능함을 안내하고 CLAUDE.md의 스크린샷 재촬영 절차를 참조한다.
