# TBM 관리자 사용 설명서 — 목차 초안

> 대상: 비개발자 관리자 (영업, 마케팅, 운영, 총무, 관리직 등)
> 원칙: 메뉴 순서가 아닌 **업무 시나리오 순서**. 각 항목은 "목적 1줄 → 스크린샷 → 번호 단계 → 주의사항" 패턴, 1작업 1페이지.
> 용어: 앱 UI 문구와 100% 동일하게 사용. "템플릿" = "양식" (UI에 혼용됨 — 문서에서는 "양식(템플릿)"으로 첫 등장 시 병기).

---

## 0. 들어가기 전에 — 핵심 개념 5분 정리

| 개념 | 한 줄 설명 |
|---|---|
| 양식(템플릿) | 주의사항·보호구(PPE)·특별 공지를 묶어둔 재사용 서식 |
| TBM 전송(세션) | 양식을 특정 날짜·그룹에 실제로 발송한 1회 건 |
| 예약중 / 전송 완료 | 예약중 = 수정·삭제 가능. 전송 완료 = 읽기 전용, Push 재발송만 가능 |
| 근로자 / 외부 근로자 | 근로자 = 조직도 소속 정규 인원. 외부 근로자 = 인증번호로 참여하는 일용직 |
| 파견 | 다른 조직·공장 소속 인원을 우리 사업장 TBM 대상으로 초대 (보라색 "파견" 배지) |
| 권한 | Admin(전권) > Manager(TBM·근로자 관리) > Worker(웹 접속 불가) |

## 1. 시작하기

### 1.1 로그인
- 이메일·비밀번호 입력, 언어 선택(한국어/English), Remember me
- 자주 보는 오류: "이메일 또는 비밀번호가 올바르지 않습니다." / "TBM 사용 계약이 없습니다." / "조직에 포함되어있지 않습니다." → 각각 대처법
- 페이지: LoginPage

### 1.2 화면 구성 익히기
- 상단 헤더: 알림 벨(30초 자동 갱신, SOS/작업중지/신규 민원/새 답글), 모두 지우기
- 사이드바: 메뉴 트리, 사용자 카드(권한 배지), 언어 전환, 로그아웃, 접기/펼치기
- 대시보드(홈): 지표 카드 4종, 빠른 작업, 알림 센터, 주간 참여율 그래프, 최근 TBM
- 페이지: HomePage, GlobalHeader, SideMenu

### 1.3 내 권한 확인하기
- 권한별 보이는 메뉴 차이표 (Admin / Manager / Worker)
- "메뉴가 안 보여요" → 권한 문제 진단 순서
- 참조: PermGuard (tbmCrud, workerInvite, adminOnly)

## 2. 매일 하는 일 — TBM 발송과 확인

### 2.1 TBM 전송 등록하기 (핵심 시나리오 ★)
- 일정 이름 → 수신 그룹 선택(전체 선택) → 전송 일시 → 서명 방식(자필/없음/Geo-Fence) → 교육 시간 반영 → 양식 선택·미리보기 → 저장
- 주의: 과거 시간 선택 불가, 기본값은 "권한 및 설정 > 기본 TBM 설정"에서 옴
- 권한: tbmCrud · 페이지: TBMSendPage (`/tbm-send`)

### 2.2 발송 예약 확인·수정하기
- 이력에서 "예약중" 건 찾기 → 전송 일시/수신 그룹/양식/수신자 수정, 삭제
- 주의: 전송 완료된 TBM은 수정·삭제 불가
- 권한: tbmCrud · 페이지: TBMSendHistoryPage, TBMSendHistoryDetailPage

### 2.3 참여 현황 확인하기
- 이력 목록: 상태/참여율/검색·정렬
- 상세: 참여 완료 / 미참여 그룹별 목록, 정규 멤버·외부 근로자 구분
- 페이지: TBMSendHistoryDetailPage (`/tbm-history/:id`)

### 2.4 미확인자에게 다시 알리기
- "Push 알림 재발송" 버튼 — 발송 후 N시간 경과해야 활성화(기본값은 설정에서 변경)
- 페이지: TBMSendHistoryDetailPage

## 3. 가끔 하는 일 — 양식과 인원 관리

### 3.1 양식(템플릿) 만들기
- 설문 소스 5종 탭: 로봇 셀 점검항목(위험성 평가) / 유해 요인별 / 설비별 / 개인 안전 보호구 / 사용자 커스텀
- 항목을 "주의사항"·"PPE" 섹션에 추가, 직접 입력, 문서 이름·특별 공지 → 양식 저장
- 권한: tbmCrud · 페이지: TBMTemplateCreatePage (`/tbm-template/create`)

### 3.2 양식 대량 등록 (Excel)
- 커스텀 탭: 양식 다운로드(CSV) → 작성 → Excel 업로드 → 선택 삭제
- 페이지: TBMTemplateCreatePage (커스텀 탭)

### 3.3 양식 수정·복원·삭제
- 목록에서 미리보기 → 수정(`?edit=`) / 삭제
- 수정 화면의 "원래대로"(원본 복구) 사용법
- 페이지: TBMTemplateListPage (`/tbm-template`)

### 3.4 근로자(정규) 관리
- 조직도 트리 탐색(회사→공장→인원, "그룹 없음")
- 인원 추가(공장 선택 시), 그룹에서 제거, 상세 보기(TBM 참여 내역)
- 권한(역할) 변경 — 본인보다 낮은 등급만 가능
- 주의: 그룹(사업장) 추가는 Admin 전용 → 설정 페이지로
- 권한: workerInvite · 페이지: UserListPage (`/users`)

### 3.5 외부 근로자(일용직) 관리
- 공장별 현황: 근무 이력·TBM 참여 내역, 인증 정보 초기화(생체/PIN)
- 기간별 TBM 참여 현황 집계
- 인증번호: 공장별 확인·갱신·일괄 갱신·일괄 출력(A4 PDF)
- 권한: workerInvite · 페이지: TemporaryWorkersPage (`/users/temporary`)

### 3.6 파견 관리
- 파견 초대: 이메일/전화 검색 → 기간·공종·메모 입력 → 초대 발송
- 파견 현황: 상태(대기중/수락됨/거절됨/취소됨/만료됨), 초대 취소, 파견 종료
- 개념 주의: 파견 ≠ 외부 근로자
- 페이지: UserListPage > 파견 관리(DispatchModal)

## 4. 문제 생겼을 때 — 민원·SOS 대응

### 4.1 SOS·긴급 상황 대응 (★ 최우선)
- 상단 SOS 카드(깜빡임) 클릭 → 상세 확인 → 조치 이력 남기기 → 조치 완료
- 알림 벨 🚨 SOS / ⚠️ 작업중지 구분
- 페이지: ComplaintsPage (`/complaints`)

### 4.2 민원 조회·필터
- 유형 5종: SOS / 작업중지 / 안전 / 건의 / TBM 이슈
- 상태 3종: 신규 → 조치 중 → 조치 완료
- 미처리만 보기, 기간·공장 필터, 검색
- 페이지: ComplaintsPage

### 4.3 민원 처리하기
- 상세 모달: 상태 변경, 조치 이력(댓글) 추가 — 댓글 등록 시 자동 "조치 중" 전환
- 일괄 조치 완료, 조치 중으로 원복
- 삭제 규칙: Admin은 SOS 외 삭제 가능, 일반 사용자는 본인 작성 건만
- 민원 등록(Admin 전용)

### 4.4 자주 묻는 질문 (FAQ)
- (CS 문의 데이터에서 추출 — 추후 채움)
- 후보: "전송 완료 TBM을 수정하고 싶어요" / "재발송 버튼이 비활성화예요" / "메뉴가 안 보여요" / "외부 근로자가 앱 인증이 안 돼요"

## 5. 관리자(Admin) 전용 — 시스템 설정

### 5.1 역할 및 권한 설정
- 권한 × 역할 매트릭스, 항목별 의미(tbmCrud, factoryCrud, workerInvite, inviteLinkCert, receiveAlert 등)
- 페이지: SettingsPage (`/settings?tab=roles`)

### 5.2 공장(사업장) 관리
- 공장 추가(최대 10개)·수정·삭제
- SOS 담당자·작업중지 관리자·관리자 지정(매니저 자동 부여)
- 지도에서 GPS 핀 찍기 + 반경 설정 → Geo-Fence 서명과 연동
- 페이지: SettingsPage (`?tab=organization`)

### 5.3 기본 TBM 설정
- 기본 서명 방식, 교육 시간 반영, 미확인자 재발송 기준 시간(1/2/4/8h)
- 미참여 알림(그룹 일수/인원 횟수), 자동 로그인, 공장별 기본 전송 시간
- 페이지: SettingsPage (`?tab=tbm-defaults`)

### 5.4 초대 및 사용자 관리
- "매니저(Manager)도 초대 가능" 설정
- 페이지: SettingsPage (`?tab=invitation`)
